定时器处理非活动连接
===============

当前版本的超时管理已经改为基于最小堆的连接定时器，用来清理长时间无读写活动的连接。

实现位置
---------------

- `timer/heap_timer.h`
- `timer/heap_timer.cpp`
- `webserver.cpp` 中 `WebServer::SubReactor`

设计思路
---------------

每个 `SubReactor` 维护一个最小堆，堆顶永远是最早到期的连接。

- 新连接注册完成后，调用 `refresh_timer(sockfd)` 加入最小堆
- 连接发生读写事件后，再次调用 `refresh_timer(sockfd)` 刷新过期时间
- 连接关闭时，调用 `remove_connection(sockfd)` 从最小堆删除
- `epoll_wait()` 的超时时间由堆顶节点决定，避免固定周期轮询
- 每轮事件处理完成后执行 `scan_timeout()`，批量关闭已经超时的连接

最小堆节点
---------------

最小堆节点只保存两项信息：

- `sockfd`：连接对应的文件描述符
- `expire`：该连接的过期时间

同时用一个哈希表维护 `sockfd -> 堆下标` 的映射，这样可以在刷新和删除时快速定位节点。

核心操作
---------------

1. 新增或刷新定时器 `add_or_update(sockfd, timeout)`

- 如果连接不在堆中，直接插入堆尾，然后上浮
- 如果连接已存在，更新过期时间，再根据大小关系执行上浮或下沉

时间复杂度：`O(log n)`

2. 删除定时器 `remove(sockfd)`

- 通过哈希表找到节点位置
- 与堆尾交换后删除
- 再对当前位置做上浮或下沉，恢复最小堆性质

时间复杂度：`O(log n)`

3. 处理超时连接 `tick(callback)`

- 反复检查堆顶
- 如果堆顶已经到期，则弹出并执行回调
- 如果堆顶未到期，则停止扫描

由于只从堆顶开始处理，所以不会遍历全部连接。

4. 获取下一次等待时间 `get_next_timeout_ms()`

- 堆为空时返回 `-1`，表示 `epoll_wait()` 可无限阻塞
- 堆顶已到期时返回 `0`
- 否则返回“堆顶过期时间 - 当前时间”

为什么使用最小堆
---------------

相较于升序链表，最小堆更适合大量连接场景下的超时管理：

- 插入和刷新成本更低，复杂度稳定在 `O(log n)`
- 堆顶就是最近超时连接，取最近超时节点成本是 `O(1)`
- 可以直接驱动 `epoll_wait()` 的超时时间，减少无意义唤醒

当前流程
---------------

1. 主 Reactor 接收连接
2. 连接分发到某个 `SubReactor`
3. `SubReactor::register_connection()` 初始化连接并加入最小堆
4. 每次读写成功后刷新该连接的超时时间
5. `SubReactor::scan_timeout()` 处理堆顶到期连接
6. 超时连接执行 `close_conn()`，释放文件描述符和相关资源
